Git console

Введение

Главное, что мы должны понять: Git - это не просто файлообменник. Это машина времени для вашего кода. Каждое "сохранение" (коммит) это не просто перезапись файла, а создание снимка всей папки проекта в определенный момент времени. Эти снимки выстраиваются в историю. Мы можем в любой момент вернуться к любому снимку, посмотреть, что изменилось, или даже создать "альтернативную версию проекта" (ветку), чтобы попробовать реализовать что-то, не ломая основной код.

Кроме возможности работать с git с помощью графического интерфейса внутри Visual Studio или Visual Studio Code работать с системой контроля версий можно с помощью консоли.

Установка

Скачать git для Windows можно тут - https://gitforwindows.org/

Во время установки нужно отметить Open Git Bash here. Это позволит открывать консоль в выбранной папке.

![[git_console_installation.png]]

Для открытия консоли в текущей папке нужно нажать правой кнопкой мыши и выбрать пункт Open Git Bash here. ![[git_open_bash_here.png]]

gitignore

Определение

Файл .gitignore (именно так как написано) содержит в себе набор папок, конкретных файлов или по расширению (*.exe), которые git должен игнорировать при добавлении в репозиторий.

Готовые шаблоны .gitignore файлов можно найти в интернете, в отличие от языка программирования и фреймворка шаблон может отличаться.

Создание gitignore

Для наших проектов (WPF .Net, он же Майкрософт) шаблон можно создать командой:

dotnet new gitignore

Данную команду нужно выполнять в консоли, находясь при этом в папке с проектом. ![[git_console_create_gitignore_file.png]] После выполнения команды в папке появится заполненный текстовый файл .gitignore.

Что может быть в gitignore

В данном текстовом файле каждое новое правило задается с новой строки.

ВСЕ ЧТО НАПИСАНО В ФАЙЛЕ - НЕ ОТСЛЕЖИВАЕТСЯ GIT

Конкретный файл.

Если мы хотим игнорировать конкретный файл, то мы должны указать полный путь до файла. Пример:

# Игнорить файл в корне
secret.txt

# Игнорить файл в конкретной папке
config/passwords.ini

Целая папка.

Для игнорирования папки и все что в ней находится нужно указать название папки и в конце строки /. Слэш важен, без него строка будет считаться файлом.

# Игнорить папку logs со всем, что внутри
logs/

# Игнорить папку с результатами сборки
bin/
obj/

Маски (Шаблоны) с помощью *.

Звёздочка * указывает на любое количество символов (по сути "написано что угодно"). 

# Игнорить ВСЕ файлы с расширением .log
*.log

# Игнорить ВСЕ файлы, чьё имя начинается с temp_
temp_*

# Игнорить ВСЕ .tmp файлы внутри папки 'data'
data/*.tmp

Исключения с помощью !.

Если нужно сделать исключение. Пример:

# Игнорить все .dll файлы...
*.dll

# ...КРОМЕ вот этой одной, она нам нужна
!libs/important.dll
