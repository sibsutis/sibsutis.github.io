Лабораторная №1, часть 1

Вступление

Данная лабораторная работа посвящена настройке рабочего окружения, знакомству с системой контроля версий Git и изучению базовых элементов разметки WPF.

Что будем делать:

Git:

  • Создание аккаунта на GitHub
  • Настройка репозитория проекта
  • Работа с .gitignore
  • Первый коммит и push

WPF:

  • Создание проекта WPF
  • Разбор основных элементов для верстки приложения
  • Фиксация внесённых изменений для каждого элемента
  • Отправка (push) в отдельную ветку feature/lab-1-basic-elements

Техническое задание

Требования к программе, необходимые для сдачи.

Легенда

Ты — новый стажёр в IT-компании. Тебе выдали пустой репозиторий. Твоя задача — не засыпать его мусором в первый же день.

Шаг 1: Создание аккаунта на GitHub

Если у тебя ещё нет аккаунта:

  1. Перейди на github.com/gitverse.ru/ любой другой гит сервер
  2. Нажми Sign up
  3. Заполни форму регистрации
  4. Подтверди email

Шаг 2: Создание репозитория

  1. На главной странице GitHub нажми New repository
  2. Заполни поля:
    • Repository name: CinemaKiosk_Фамилия (например, CinemaKiosk_Ivanov)
    • Description: Проект киоска бронирования билетов
    • Visibility: Private
  3. НЕ ставь галочки на "Add README" и "Add .gitignore" — сделаем это вручную
  4. Нажми Create repository

Шаг 3: Создание Solution в Visual Studio

  1. Открой Visual Studio
  2. Создать новый проект → Приложение WPF (Майкрософт)
  3. Укажи путь к папке, которую будешь связывать с репозиторием
  4. Название проекта: CinemaKiosk

Шаг 4: Настройка .gitignore

Это критически важный шаг. Файл .gitignore указывает Git, какие файлы НЕ нужно отслеживать.

  1. В корне проекта создай файл .gitignore
  2. Скопируй содержимое из gitignore.io.
Пример файла .gitignore 
# Visual Studio
.vs/
bin/
obj/
*.user
*.suo
*.cache

# Build results
[Dd]ebug/
[Rr]elease/

# NuGet
packages/
*.nupkg

Вы можете использовать команду для cmd внутри папки с проектом:

dotnet new gitignore

Данная команда сгенерирует очень объемный файл для игнорирования файлов, предпочтительнее.

Автоматическая пересдача

Если в репозитории окажутся папки /bin, /obj или .vs - это бан.

Шаг 6: Первый коммит и push

Открой терминал (Git Bash или PowerShell) в папке проекта:

# Инициализация репозитория локально
git init

# Добавление всех файлов которые изменились 
# Создание файла тоже считается
git add .

# Первый коммит
git commit -m "Initial commit: project setup"

# Связь с удалённым репозиторием
# удалённых репозиториев может быть несколько
# одной командой можно отправить изменения во все 
git remote add origin https://github.com/ТВОЙ_ЛОГИН/CinemaKiosk_Фамилия.git

# Push в main
git branch -M main
git push -u origin main
Данный раздел обязателен к выполнению для групп ТТП

Шаг 7: Приглашение преподавателя

  1. Перейди в репозиторий на GitHub
  2. Settings → Collaborators
  3. Нажми Add people
  4. Введи логин преподавателя
  5. Отправь приглашение

Проверка перед сдачей

Убедитесь, что:

Репозиторий создан и приватный .gitignore настроен правильно В репозитории НЕТ папок bin/, obj/, .vs/ Есть файл README.md с твоими данными Преподаватель получил приглашение в Collaborators
